## 实现步骤

### 步骤 1：设置 Slack 集成（10 分钟）

1. 创建 Slack webhook：
   - 访问 [Slack API Apps](https://api.slack.com/apps)
   - 创建新应用 → "从头开始"（[创建应用指南](https://api.slack.com/authentication/basics#creating)）
   - 应用名称："MCP Course Notifications"
   - 选择你的工作区
   - 进入"功能" → "[传入 Webhooks](https://api.slack.com/messaging/webhooks)"
   - [激活传入 webhooks](https://api.slack.com/messaging/webhooks#enable_webhooks)
   - 点击"添加新的 Webhook 到工作区"
   - 选择频道并授权（[Webhook 设置指南](https://api.slack.com/messaging/webhooks#getting_started)）
   - 复制 webhook URL

2. 测试 webhook 是否工作（遵循 [webhook 发布示例](https://api.slack.com/messaging/webhooks#posting_with_webhooks)）：
   ```bash
   curl -X POST -H 'Content-type: application/json' \
     --data '{"text":"Hello from MCP Course!"}' \
     YOUR_WEBHOOK_URL
   ```

3. 设置环境变量：
   ```bash
   export SLACK_WEBHOOK_URL="https://hooks.slack.com/services/YOUR/WEBHOOK/URL"
   ```

   **⚠️ 安全提示**：webhook URL 是一个敏感的密钥，授予向你的 Slack 频道发布消息的权限。始终：
   - 将其存储为环境变量，永远不要在代码中硬编码
   - 永远不要将 webhook URL 提交到版本控制（添加到 .gitignore）
   - 像对待密码一样对待它 - 任何拥有此 URL 的人都可以向你的频道发送消息

> [!WARNING]
> **安全警报**：Webhook URL 是敏感凭证！任何拥有你的 webhook URL 的人都可以向你的 Slack 频道发送消息。始终将它们存储为环境变量，永远不要提交到版本控制。

### 步骤 3：创建格式化提示词（15 分钟）

接下来，你将向服务器添加 MCP 提示词——这就是魔法发生的地方！这些提示词将与 Claude 协作，自动将你的 GitHub webhook 数据格式化为结构良好的 Slack 消息。记住模块 1 中提到的，提示词提供了 Claude 可以一致使用的可重用指令。

实现两个生成 Slack 格式化消息的提示词：

1. **`format_ci_failure_alert`**：
   ```python

       """为 CI/CD 失败创建 Slack 警报。"""
       return """将此 GitHub Actions 失败格式化为 Slack 消息：

   使用此模板：
   :rotating_light: *CI 失败警报* :rotating_light:
   
   CI 工作流失败：
   *工作流*：workflow_name
   *分支*：branch_name
   *状态*：失败
   *查看详情*：<LOGS_LINK|查看日志>
   
   请检查日志并解决任何问题。
   
   使用 Slack markdown 格式，保持简洁以便团队快速扫描。"""
   ```

## 常见问题

### Webhook URL 问题
- 验证环境变量设置正确
- 在集成之前直接使用 curl 测试 webhook
- 确保 Slack 应用具有适当的权限

### 消息格式化
- [Slack markdown](https://api.slack.com/reference/surfaces/formatting) 与 GitHub markdown 不同
- **重要**：使用 `*文本*` 表示粗体（不是 `**文本**`）
- 在 webhook 负载中包含 `"mrkdwn": true` 以实现正确的格式化
- 在自动化之前手动测试消息格式
- 正确处理提交消息中的特殊字符（[格式化参考](https://api.slack.com/reference/surfaces/formatting#escaping)）

### 网络错误
- 向 webhook 请求添加基本的超时处理（[webhook 错误处理](https://api.slack.com/messaging/webhooks#handling_errors)）
- 从工具返回有意义的错误消息
- 如果请求失败，检查互联网连接
