本模块将演示一个完整的自动化工作流闭环：

## 实现步骤

### 步骤 1：设置 Slack 集成 (约 10 分钟)

1. 创建一个 Slack Webhook：
   - 访问 [Slack API 应用](https://api.slack.com/apps)
   - 创建新应用 → "From scratch" ([创建应用指南](https://api.slack.com/authentication/basics#creating))
   - 应用名称："MCP Course Notifications"
   - 选择您的工作区
   - 进入"功能" → "[Incoming Webhooks](https://api.slack.com/messaging/webhooks)"
   - [激活传入 Webhooks](https://api.slack.com/messaging/webhooks#enable_webhooks)
   - 点击"Add New Webhook to Workspace"
   - 选择一个频道并授权 ([Webhook 设置指南](https://api.slack.com/messaging/webhooks#getting_started))
   - **复制生成的 Webhook URL**

2. 测试 Webhook 是否正常工作 (参考 [Webhook 发送示例](https://api.slack.com/messaging/webhooks#posting_with_webhooks))：
   ```bash
   curl -X POST -H 'Content-type: application/json' \
     --data '{"text":"Hello from MCP Course!"}' \
     YOUR_WEBHOOK_URL
   ```

3. 设置环境变量：
   ```bash
   export SLACK_WEBHOOK_URL="https://hooks.slack.com/services/YOUR/WEBHOOK/URL"
   ```

   > [!WARNING]
   > **⚠️ 安全警报：Webhook URL 是敏感凭证！**
   > 任何拥有您 Webhook URL 的人都可以向您的 Slack 频道发送消息。请务必：
   > -   **存储为环境变量**：绝对不要在代码中硬编码。
   > -   **切勿提交到版本控制**：确保已将其添加到 `.gitignore` 文件中。
   > -   **视同密码**：像对待密码一样保护它。

## 常见问题解答

### Webhook URL 问题
- 验证环境变量是否已正确设置。
- 在集成前，直接使用 curl 测试 Webhook URL 是否有效。
- 确保您的 Slack 应用具有相应的发布权限。

### 消息格式化
- [Slack Markdown](https://api.slack.com/reference/surfaces/formatting) 与 GitHub Markdown 不同。
- **重要**：使用 `*文本*` 表示粗体 (而非 `**文本**`)。
- 在 Webhook 的 JSON 负载中包含 `"mrkdwn": true` 以确保格式正确渲染。
- 在自动化之前，手动测试消息格式。
- 正确处理提交消息中的特殊字符 ([格式化参考](https://api.slack.com/reference/surfaces/formatting#escaping))。

### 网络错误
- 为 Webhook 请求添加基本的超时处理 ([错误处理指南](https://api.slack.com/messaging/webhooks#handling_errors))。
- 确保您的工具能返回有意义的错误消息。
- 如果请求失败，请检查您的网络连接。
