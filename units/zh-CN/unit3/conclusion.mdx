## 你学到的关键 MCP 概念

### MCP 原语
- **Tools（工具）**：用于数据访问和外部 API 调用
- **Prompts（提示）**：用于一致的工作流指导和格式化
- **集成模式**：工具和提示如何协同工作

### 架构模式
- **关注点分离**：MCP 服务器 vs webhook 服务器
- **基于文件的事件存储**：简单、可靠、可测试
- **Claude 作为智能层**：从原始数据做出决策

### 开发最佳实践
- **错误处理**：即使失败也返回结构化 JSON
- **安全性**：使用环境变量存储敏感凭证
- **测试**：验证脚本和手动测试工作流

## 关键要点

> [!TIP]
> **MCP 理念**：最有效的 MCP 服务器不会试图变得聪明 - 它们为 Claude 提供丰富的结构化数据，让 Claude 的智能来完成繁重的工作。这使你的代码更简单、更灵活。

### 技术见解
- **简单即强大**：JSON 文件存储可以处理许多用例
- **Claude 作为协调者**：让 Claude 在你的工具之间进行协调
- **提示保证一致性**：使用提示确保可靠的输出格式

### 开发见解
- **从小处着手**：一次构建一个工具，彻底测试
- **以工作流思考**：设计能够良好协同工作的工具
- **为人类规划**：你的自动化应该帮助团队，而不是取代他们
