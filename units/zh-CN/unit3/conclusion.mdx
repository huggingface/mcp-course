## 您学到的关键 MCP 概念

### MCP 原语
-   **工具 (Tools)**：用于数据访问和外部 API 调用
-   **提示词 (Prompts)**：用于一致的工作流指导和格式化
-   **集成模式**：工具和提示词如何协同工作

### 架构模式
-   **关注点分离**：MCP 服务器 vs. Webhook 服务器
-   **基于文件的事件存储**：简单、可靠、可测试
-   **AI 作为智能层**：从原始数据做出决策

### 开发最佳实践
-   **错误处理**：即使失败，也返回结构化的 JSON
-   **安全性**：使用环境变量存储敏感的凭证
-   **测试**：验证脚本和手动测试工作流

## 关键要点

> [!TIP]
> **MCP 理念**：最有效的 MCP 服务器并不会试图变得“聪明”——它们为 AI 提供丰富的结构化数据，然后让 AI 的智能来完成繁重的工作。这使得您的代码更简单、更灵活。

### 技术见解
-   **简单即是强大**：基于 JSON 文件的存储可以处理许多用例
-   **AI 作为协调者**：让 AI 在您的工具之间进行协调
-   **提示词保证一致性**：使用提示词来确保可靠的输出格式

### 开发见解
-   **从小处着手**：一次构建一个工具，并进行彻底的测试
-   **以工作流的视角思考**：设计能够良好协同工作的工具
-   **为人类而规划**：您的自动化应该帮助团队，而不是取代他们
