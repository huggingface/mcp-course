这是你在 CodeCraft Studios 的第一周，你正在目睹一些让每个开发者都感到头疼的事情。团队的拉取请求看起来是这样的：

- "stuff"（一些东西）

与此同时，代码审查积压越来越多，因为审查者无法理解发生了什么变化或为什么要这样做。后端团队的 Sarah 花了 30 分钟试图弄清楚"various improvements"（各种改进）实际上是什么意思，而前端的 Mike 不得不翻阅 47 个文件来理解一个"small fix"（小修复）。

团队知道他们需要更好的 PR 描述，但每个人都忙于交付功能，没时间写详细的说明。他们需要一个既能提供帮助又不会拖慢进度的解决方案。

**你的任务**：构建一个智能 PR 代理，自动分析代码变更并建议有用的描述。

## 概述

你的 PR 代理将使用 MCP 开发的一个关键原则来解决 CodeCraft Studios 的问题：不是硬编码关于什么是好 PR 的僵化规则，而是向 Claude 提供原始 git 数据，让它智能地建议适当的描述。

这种方法有效是因为：
- **灵活分析**：Claude 可以理解简单规则遗漏的上下文
- **自然语言**：建议感觉像人类，而不是机器人
- **适应性强**：适用于任何代码库或编码风格

你将实现三个基本工具，为整个自动化系统建立模式：

1. **analyze_file_changes** - 检索 git diff 信息和更改的文件（数据收集）
2. **get_pr_templates** - 列出可用的 PR 模板（资源管理）
3. **suggest_template** - 允许 Claude 推荐最合适的模板（智能决策）

## 常见模式

### 工具实现模式

```python
@mcp.tool()
async def tool_name(param1: str, param2: bool = True) -> str:
    """Claude 的工具描述。
    
    Args:
        param1: 参数描述
        param2: 带默认值的可选参数
    """
    # 你的实现
    result = {"key": "value"}
    return json.dumps(result)
```

### 错误处理

始终优雅地处理潜在错误：

```python
try:
    result = subprocess.run(["git", "diff"], capture_output=True, text=True)
    return json.dumps({"output": result.stdout})
except Exception as e:
    return json.dumps({"error": str(e)})
```

> [!WARNING]
> **错误处理**：始终从你的工具返回有效的 JSON，即使是错误。Claude 需要结构化数据来理解出了什么问题，并向用户提供有用的响应。

### 处理大型输出（关键学习时刻！）

> [!WARNING]
> **真实世界约束**：MCP 工具每个响应有 25,000 个令牌的限制。大型 git 差异很容易超过这个限制 10 倍或更多！这是生产 MCP 开发的关键课程。

在实现 `analyze_file_changes` 时，你可能会遇到这个错误：
```
Error: MCP tool response (262521 tokens) exceeds maximum allowed tokens (25000)
```

**为什么会发生这种情况：**
- 单个文件更改可能有数千行
- 企业仓库通常有大规模重构
- Git 差异默认包含完整上下文
- JSON 编码增加了开销

这教会我们一个重要原则：**始终在设计工具时考虑输出限制**。这是解决方案：

```python
@mcp.tool()
async def analyze_file_changes(base_branch: str = "main", 
                              include_diff: bool = True,
                              max_diff_lines: int = 500) -> str:
    """使用智能输出限制分析文件更改。
    
    Args:
        base_branch: 要比较的分支
        include_diff: 是否包含实际差异
        max_diff_lines: 要包含的最大差异行数（默认 500）
    """
    try:
        # 获取差异
        result = subprocess.run(
            ["git", "diff", f"{base_branch}...HEAD"],
            capture_output=True, 
            text=True
        )
        
        diff_output = result.stdout
        diff_lines = diff_output.split('\n')
        
        # 如果需要，智能截断
        if len(diff_lines) > max_diff_lines:
            truncated_diff = '\n'.join(diff_lines[:max_diff_lines])
            truncated_diff += f"\n\n... 输出已截断。显示 {len(diff_lines)} 行中的 {max_diff_lines} 行 ..."
            diff_output = truncated_diff
        
        # 获取摘要统计信息
        stats_result = subprocess.run(
            ["git", "diff", "--stat", f"{base_branch}...HEAD"],
            capture_output=True,
            text=True
        )
        
        return json.dumps({
            "stats": stats_result.stdout,
            "total_lines": len(diff_lines),
            "diff": diff_output if include_diff else "使用 include_diff=true 查看差异",
            "files_changed": self._get_changed_files(base_branch)
        })
        
    except Exception as e:
        return json.dumps({"error": str(e)})
```

**大型输出的最佳实践：**
1. **实现分页**：将大型结果分成页面
2. **添加过滤选项**：让用户请求特定文件或目录
3. **首先提供摘要**：在完整内容之前返回统计信息
4. **使用渐进式披露**：从高级信息开始，允许深入了解
5. **设置合理的默认值**：默认为适用于大多数情况的合理限制

## 工作目录注意事项

默认情况下，MCP 服务器在其安装目录中运行命令，而不是在 Claude 的当前工作目录中。这意味着你的 git 命令可能会分析错误的仓库！

为了解决这个问题，MCP 提供了 [roots](https://modelcontextprotocol.io/docs/concepts/roots) - 一种让客户端通知服务器相关目录的方式。Claude Code 自动提供其工作目录作为 root。

以下是如何在你的工具中访问它：

```python
@mcp.tool()
async def analyze_file_changes(...):
    # 从 roots 获取 Claude 的工作目录
    context = mcp.get_context()
    roots_result = await context.session.list_roots()
    
    # 从 FileUrl 对象中提取路径
    working_dir = roots_result.roots[0].uri.path
    
    # 用于所有 git 命令
    result = subprocess.run(
        ["git", "diff", "--name-status"],
        capture_output=True,
        text=True,
        cwd=working_dir  # 在 Claude 的目录中运行！
    )
```

这确保你的工具在 Claude 实际工作的仓库上操作，而不是 MCP 服务器的安装位置。

## 故障排除

- **导入错误**：确保你已运行 `uv sync`
- **Git 错误**：确保你在 git 仓库中
- **无输出**：MCP 服务器通过 stdio 通信 - 使用 Claude Desktop 测试
- **JSON 错误**：所有工具必须返回有效的 JSON 字符串
- **超过令牌限制**：这对于大型差异是预期的！如上所示实现输出限制
- **"响应太大"错误**：添加 `max_diff_lines` 参数或设置 `include_diff=false`
- **Git 命令在错误的目录中运行**：MCP 服务器默认在其安装目录中运行，而不是 Claude 的工作目录。要解决这个问题，使用 [MCP roots](https://modelcontextprotocol.io/docs/concepts/roots) 访问 Claude 的当前目录：
  ```python
  # 从 roots 获取 Claude 的工作目录
  context = mcp.get_context()
  roots_result = await context.session.list_roots()
  working_dir = roots_result.roots[0].uri.path  # FileUrl 对象有 .path 属性
  
  # 在 subprocess 调用中使用它
  subprocess.run(["git", "diff"], cwd=working_dir)
  ```
  Claude Code 自动提供其工作目录作为 root，允许你的 MCP 服务器在正确的位置操作。
