# 项目基础设定

在本节中，我们将为我们的“PR Agent” (Pull Request Agent) 搭建好开发环境。

> [!TIP]
> 我们将采用现代化的 Python 工具链，使用 `uv` 来管理依赖项并创建必要的配置文件。如果您对 `uv` 不熟悉，可以查阅[官方文档](https://docs.astral.sh/uv/)来了解更多信息。

## 项目结构

让我们从克隆项目目录开始，并熟悉其文件结构：

```bash
git clone https://huggingface.co/spaces/mcp-course/tag-this-repo
```

最终，我们的项目结构将如下所示：

```
hf-pr-agent/
├── mcp_server.py              # 核心 MCP 服务器及标签工具
├── app.py                     # FastAPI Webhook 监听器和 Agent 
├── requirements.txt           # Python 依赖项
├── pyproject.toml             # 项目配置
├── env.example                # 环境变量模板
├── cleanup.py                 # 用于开发的实用工具
```

## 依赖项与配置

现在，我们来逐步了解项目的依赖项和配置。

### 1. Python 项目配置

我们将使用 `uv` 来管理 `pyproject.toml` 文件，以定义我们的项目。

> [!TIP]
> 如果您尚未安装 `uv`，可以按照[此处的说明](https://docs.astral.sh/uv/getting-started/installation/)进行安装。

```toml
[project]
name = "mcp-course-unit3-example"
version = "0.1.0"
description = "一个用于处理 Hugging Face Hub 讨论区 Webhook 的 FastAPI 和 Gradio 应用"
readme = "README.md"
requires-python = ">=3.11"
dependencies = [
    "fastapi>=0.104.0",
    "uvicorn[standard]>=0.24.0",
    "gradio>=4.0.0",
    "huggingface-hub[mcp]>=0.32.0",
    "pydantic>=2.0.0",
    "python-multipart>=0.0.6",
    "requests>=2.31.0",
    "python-dotenv>=1.0.0",
    "fastmcp>=2.0.0",
]

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[tool.hatch.build.targets.wheel]
packages = ["src"]
```

为了与各种部署平台保持兼容，我们在 `requirements.txt` 文件中也保留了相同的内容。

要创建虚拟环境，请运行：

```bash
uv venv
source .venv/bin/activate # 在 Windows 上，请使用 .venv/Scripts/activate
```

要安装依赖项，请运行：

```bash
uv sync
```

### 2. 环境配置

创建一个名为 `env.example` 的文件来记录所需的环境变量：

```bash
# Hugging Face API 令牌（必需）
# 从 https://huggingface.co/settings/tokens 获取
HF_TOKEN=hf_your_token_here

# Webhook 密钥（生产环境必需）
# 使用一个强随机字符串
WEBHOOK_SECRET=your-webhook-secret-here

# Agent 使用的模型（可选）
HF_MODEL=owner/model

# MCP 的提供商（可选）
HF_PROVIDER=huggingface
```

您需要从[此处](https://huggingface.co/settings/tokens)获取您的 Hugging Face API 令牌。

您还需要生成一个 Webhook 密钥。可以通过运行以下命令来完成：

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

然后，您需要根据 `env.example` 文件的内容，将生成的 Webhook 密钥添加到您的 `.env` 文件中。

> [!WARNING]
> **保护好您的 `.env` 文件**
> `env.example` 只是一个模板。您需要创建一个名为 `.env` 的新文件，并将您的实际密钥填入其中。切勿将 `.env` 文件提交到版本控制系统。请务必将 `.env` 添加到您的 `.gitignore` 文件中，以防止意外泄露敏感信息。
