# Gradio 作为 MCP 客户端

在上一节中，我们探讨了如何使用 Gradio 创建 MCP 服务器，并使用 MCP 客户端连接到它。在本节中，我们将反过来，探索如何使用 Gradio 作为 MCP 客户端来连接到 MCP 服务器。

> [!TIP]
> Gradio 最适合用于创建 UI 客户端和 MCP 服务器，但它也可以被用作一个 MCP 客户端，并将其功能以 UI 的形式暴露出来。

我们将连接到一个类似于上一节创建的、但功能更丰富的 MCP 服务器，并利用它来回答问题。

## Gradio 中的 MCP 客户端

### 连接到示例 MCP 服务器

让我们连接到一个已经在 Hugging Face 上运行的示例 MCP 服务器。我们将使用[这个服务器](https://huggingface.co/spaces/abidlabs/mcp-tool-http)作为示例。这是一个包含了多种 MCP 工具的 Space。

```python
from smolagents.mcp_client import MCPClient

with MCPClient(
    {"url": "https://abidlabs-mcp-tool-http.hf.space/gradio_api/mcp/sse", "transport": "sse",}
) as tools:
    # 打印出远程服务器上可用的工具
    print("\n".join(f"{t.name}: {t.description}" for t in tools))

```

### 从 Gradio 连接到 MCP 服务器

很好，现在您已经连接到了一个示例 MCP 服务器。接下来，您需要在您的示例应用中使用它。

首先，如果尚未安装，我们需要安装 `smolagents`、`gradio` 和 `mcp-client` 库：

```bash
pip install "smolagents[mcp]" "gradio[mcp]" mcp fastmcp
```

现在，我们可以导入必要的库，并创建一个简单的 Gradio 界面，通过 MCP 客户端连接到 MCP 服务器。

```python
import gradio as gr
import os

from mcp import StdioServerParameters
from smolagents import InferenceClientModel, CodeAgent, ToolCollection, MCPClient
```

接下来，我们将连接到 MCP 服务器，并获取可用于回答问题的工具。

```python
mcp_client = MCPClient(
    {"url": "https://abidlabs-mcp-tool-http.hf.space/gradio_api/mcp/sse", "transport": "sse",} # 这是我们在上一节中创建的 MCP 客户端
)
tools = mcp_client.get_tools()
```

现在我们有了工具，可以创建一个简单的 Agent 来使用这些工具回答问题。目前，我们只使用 `smolagents` 提供的 `InferenceClientModel` 和默认模型。

很重要的一点是，您需要将您的 `api_key` 传递给 `InferenceClientModel`。您可以从您的 Hugging Face 账户访问令牌中获取。[点击这里查看详情](https://huggingface.co/docs/hub/en/security-tokens)，并使用环境变量 `HF_TOKEN` 来设置您的访问令牌。

```python
model = InferenceClientModel(token=os.getenv("HF_TOKEN"))
agent = CodeAgent(tools=[*tools], model=model)
```

现在，我们可以创建一个简单的 Gradio 界面，使用这个 Agent 来回答问题。

```python
demo = gr.ChatInterface(
    fn=lambda message, history: str(agent.run(message)),
    type="messages",
    examples=["68 的质因数是什么？"],
    title="使用 MCP 工具的 Agent ",
    description="这是一个使用 MCP 工具来回答问题的简单 Agent 。"
)

demo.launch()
```

就是这样！我们创建了一个简单的 Gradio 界面，它使用 MCP 客户端连接到 MCP 服务器，并能回答问题。

## 完整示例

以下是在 Gradio 中使用 MCP 客户端的完整示例代码：

```python
import gradio as gr
import os

from smolagents import InferenceClientModel, CodeAgent, MCPClient


try:
    mcp_client = MCPClient(
        {"url": "https://abidlabs-mcp-tool-http.hf.space/gradio_api/mcp/sse", "transport": "sse",}
    )
    tools = mcp_client.get_tools()

    model = InferenceClientModel(token=os.getenv("HUGGINGFACE_API_TOKEN"))
    agent = CodeAgent(tools=[*tools], model=model, additional_authorized_imports=["json", "ast", "urllib", "base64"])

    demo = gr.ChatInterface(
        fn=lambda message, history: str(agent.run(message)),
        type="messages",
        examples=["分析以下文本的情感 '这太棒了'"],
        title="使用 MCP 工具的 Agent ",
        description="这是一个使用 MCP 工具来回答问题的简单 Agent 。",
    )

    demo.launch()
finally:
    mcp_client.disconnect()
```

您会注意到我们在 `finally` 代码块中关闭了 MCP 客户端的连接。这一点很重要，因为 MCP 客户端是一个长期存在的对象，需要在程序退出时正确关闭。

## 部署到 Hugging Face Spaces

为了让其他人可以使用您的服务器，您可以像上一节一样，将其部署到 Hugging Face Spaces。
要将您的 Gradio MCP 客户端部署到 Hugging Face Spaces，请执行以下步骤：

1.  在 Hugging Face 上创建一个新的 Space：
    -   访问 huggingface.co/spaces
    -   点击 “Create new Space”
    -   选择 “Gradio” 作为 SDK
    -   为您的 Space 命名 (例如，“mcp-client”)

2.  在您的代码中更新 MCP 服务器的 URL：

```python
mcp_client = MCPClient(
    {"url": "https://abidlabs-mcp-tool-http.hf.space/gradio_api/mcp/sse", "transport": "sse"}
)
```

3.  创建一个 `requirements.txt` 文件：
```txt
gradio[mcp]
smolagents[mcp]
```

4.  将您的代码推送到 Space：
```bash
git init
git add app.py requirements.txt
git commit -m "Initial commit"
git remote add origin https://huggingface.co/spaces/YOUR_USERNAME/mcp-client
git push -u origin main
```

注意：在添加远程仓库时，请参考 [password-git-deprecation](https://huggingface.co/blog/password-git-deprecation) 博文，使用访问令牌进行身份验证。
