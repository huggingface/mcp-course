# 构建 MCP 客户端

在本节中，我们将使用不同的编程语言创建能够与 MCP 服务器交互的客户端。我们将分别实现一个使用 HuggingFace.js 的 JavaScript 客户端和一个使用 smol-agents 的 Python 客户端。

## 配置 MCP 客户端

要有效地部署 MCP 服务器和客户端，正确的配置至关重要。MCP 规范仍在不断发展，因此配置方法将来可能会发生变化。我们将重点关注当前推荐的最佳实践。

### MCP 配置文件

MCP 宿主使用配置文件来管理服务器连接。这些文件定义了哪些服务器可用以及如何连接到它们。

这些配置文件非常简单直观，并且在主流的 MCP 宿主之间保持着良好的一致性。

#### `mcp.json` 结构

MCP 的标准配置文件名为 `mcp.json`。以下是其基本结构：

```json
{
  "servers": [
    {
      "name": "MCP Server",
      "transport": {
        "type": "sse",
        "url": "http://localhost:7860/gradio_api/mcp/sse"
      }
    }
  ]
}
```

在此示例中，我们配置了一个使用 SSE 传输的服务器，它将连接到运行在 7860 端口的本地 Gradio 服务器。

> [!TIP]
> 我们通过 SSE 传输方式连接到 Gradio 应用，是因为我们假设该 Gradio 应用运行在一台远程服务器上。然而，如果您想连接到的是一个本地脚本，使用 `stdio` 传输会是更好的选择。

#### HTTP+SSE 传输配置

对于使用 HTTP+SSE 传输的远程服务器，配置中需要包含服务器的 URL：

```json
{
  "servers": [
    {
      "name": "Remote MCP Server",
      "transport": {
        "type": "sse",
        "url": "https://example.com/gradio_api/mcp/sse"
      }
    }
  ]
}
```

此配置允许您的 UI 客户端使用 MCP 协议与 Gradio MCP 服务器进行通信，从而实现前端与 MCP 服务之间的无缝集成。

## 配置 UI MCP 客户端

在使用 Gradio MCP 服务器时，您可以配置 UI 客户端通过 MCP 协议连接到该服务器。以下是具体的设置方法：

### 基本配置

创建一个名为 `config.json` 的新文件，并包含以下配置：

```json
{
  "mcpServers": {
    "mcp": {
      "url": "http://localhost:7860/gradio_api/mcp/sse"
    }
  }
}
```

此配置允许您的 UI 客户端使用 MCP 协议与 Gradio MCP 服务器进行通信，实现前端与 MCP 服务之间的无缝集成。

## 在 Cursor IDE 中配置 MCP 客户端

Cursor 提供了内置的 MCP 支持，允许您将已部署的 MCP 服务器直接连接到您的开发环境中。

### 配置方法

打开 Cursor 设置（`Ctrl + Shift + J` / `Cmd + Shift + J`）→ **工具与集成** 标签 → **添加自定义 MCP**：

**macOS：**
```json
{
  "mcpServers": {
    "sentiment-analysis": {
      "command": "npx",
      "args": [
        "-y", 
        "mcp-remote", 
        "https://YOURUSENAME-mcp-sentiment.hf.space/gradio_api/mcp/sse", 
        "--transport", 
        "sse-only"
      ]
    }
  }
}
```

**Windows：**
```json
{
  "mcpServers": {
    "sentiment-analysis": {
      "command": "cmd",
      "args": [
        "/c", 
        "npx", 
        "-y", 
        "mcp-remote", 
        "https://YOUR-USERNAME-mcp-sentiment.hf.space/gradio_api/mcp/sse", 
        "--transport", 
        "sse-only"
      ]
    }
  }
}
```

### 为什么使用 `mcp-remote`

> **注意**：截至 2025 年中期，Cursor 已支持通过 HTTP+SSE 和 OAuth 直接进行远程 MCP 连接。除非您使用的是旧版设置或遇到特定的兼容性问题，否则您可能不再需要 `mcp-remote`。

早期版本的 MCP 客户端 (包括 Cursor) 仅支持通过 `stdio` 传输的本地服务器，缺乏对带身份验证的远程服务器的支持。`mcp-remote` 工具作为一种解决方案应运而生，它：

-   在您的本地机器上运行
-   在 Cursor 和远程 MCP 服务器之间建立一个桥梁
-   隐式地处理传输和身份验证
-   使用您所熟悉的配置文件格式

虽然这在某些边缘情况下仍然有用，但 Cursor 现在已原生支持远程 MCP 集成。您可以像这样直接配置远程服务器：

```json
{
  "mcpServers": {
    "my-server": {
      "url": "https://your-mcp-server.hf.space/gradio_api/mcp/sse"
    }
  }
}
```
> 请参阅 [Cursor 官方文档](https://docs.cursor.com/context/mcp) 获取最新的设置说明。

配置完成后，您可以要求 Cursor 使用您的情感分析工具来执行任务，例如分析代码注释、用户反馈或拉取请求 (Pull Request, PR) 的描述。
