# 使用 AMD NPU 与 iGPU 加速的本地 Tiny Agents

在本节中，我们将向您展示如何利用 AMD 神经处理单元 (NPU) 和集成 GPU (iGPU) 来加速我们的端到端 Tiny Agents 应用。随后，我们将通过提供对本地文件的访问权限并创建一个能处理敏感信息的助手，来增强我们的应用，以确保最大程度的隐私保护。

为了实现这一目标，我们将使用 Lemonade Server，这是一个用于在本地运行模型并支持 NPU 和 iGPU 加速的工具。

## 设置
### 设置 Lemonade Server

您可以在 Windows 和 Linux 上安装 Lemonade Server。更多文档可以在 [lemonade-server.ai](https://lemonade-server.ai/) 找到。

<hfoptions id="local-models">
<hfoption id="windows">

要在 Windows 上安装 Lemonade Server，只需在[此处](https://github.com/lemonade-sdk/lemonade/releases/latest/download/Lemonade_Server_Installer.exe)下载并运行最新的安装程序即可。

Lemonade Server 在 Windows x86/x64 平台上支持所有平台和引擎的 CPU 推理。GPU 加速通过使用 Vulkan 的 `llamacpp` 引擎启用，重点支持 AMD Ryzen™ AI 7000/8000/300 系列和 AMD Radeon™ 7000/9000 系列。对于 NPU 加速，ONNX Runtime GenAI (OGA) 引擎支持 AMD Ryzen™ AI 300 系列设备。

安装 Lemonade Server 后，您可以通过点击桌面上的 `Lemonade` 图标来启动它。

</hfoption>
<hfoption id="linux">

要在 Linux 上安装 Lemonade，首先请创建并激活一个虚拟环境 (venv)：

> [!TIP]
> 如果您没有安装 `uv`，可以按照[此处的说明](https://docs.astral.sh/uv/getting-started/installation/)进行安装。

```bash
uv venv --python 3.11
source .venv/bin/activate
```

然后，安装 `lemonade-sdk` 包：
```bash
uv pip install lemonade-sdk==8.0.3
```

或者，您也可以通过克隆仓库并从源代码构建包来安装：
```bash
git clone https://github.com/lemonade-sdk/lemonade-sdk.git
cd lemonade-sdk
pip install -e .
```

安装完成后，您可以通过运行以下命令来启动 Lemonade：

```bash
lemonade-server-dev serve
```

Lemonade Server 在 Windows x86/x64 平台上支持所有平台和引擎的 CPU 推理。GPU 加速通过 `llamacpp` 引擎 (Vulkan) 启用，重点支持 AMD Ryzen™ AI 7000/8000/300 系列和 Radeon™ 7000/9000 系列。

> [!TIP]
> *NPU 加速目前仅适用于 Windows 上的 AMD Ryzen™ AI 300 系列。*

### Tiny Agents 和 NPX 设置

本课程的这一部分假设您已经安装了 `npx` 和 `Tiny Agents`。如果还没有，请参考课程的 [Tiny Agents](https://huggingface.co/learn/mcp-course/en/unit2/tiny-agents) 部分。请确保使用 `huggingface_hub[mcp]==0.33.2`。

## 创建一个在本地处理敏感信息的助手

![Lemonade Server 界面](https://raw.githubusercontent.com/lemonade-sdk/assets/refs/heads/main/huggingface_course/hf_lemonade.png)

现在，让我们通过启用对本地文件的访问权限，并引入一个完全在设备上处理敏感信息的助手，来增强我们的端到端应用。具体来说，这个助手将帮助我们评估候选人的简历，并在招聘过程中辅助决策——同时确保数据的私密性和安全性。

为此，我们将使用 [Desktop Commander](https://github.com/wonderwhy-er/DesktopCommanderMCP) MCP 服务器，它允许您在本地机器上运行命令，并提供全面的文件系统访问、终端控制和代码编辑功能。

让我们用一个基本的 Tiny Agent 来设置一个项目。

```bash
mkdir file-assistant
cd file-assistant
```

然后在 `file-assistant` 文件夹中创建一个新的 `agent.json` 文件。

<hfoptions id="agent-file-config">
<hfoption id="windows">

```json
{
  "model": "user.jan-nano",
  "endpointUrl": "http://localhost:8000/api/",
  "servers": [
    {
      "type": "stdio",
      "command": "C:\\Program Files\\nodejs\\npx.cmd",
      "args": [
        "-y",
        "@wonderwhy-er/desktop-commander"
      ]
    }
  ]
}
```

</hfoption>
<hfoption id="linux">

```json
{
  "model": "user.jan-nano",
  "endpointUrl": "http://localhost:8000/api/",
  "servers": [
    {
      "type": "stdio",
      "command": "npx",
      "args": [
        "-y",
        "@wonderwhy-er/desktop-commander"
      ]
    }
  ]
}
```

</hfoption>
</hfoptions>

最后，我们需要下载 `Jan Nano` 模型。您可以通过访问 http://localhost:8000/#model-management，点击 `Add a Model` 并提供以下信息来完成：

```
Model Name: user.jan-nano
Checkpoint: Menlo/Jan-nano-gguf:jan-nano-4b-Q4_0.gguf
Recipe: llamacpp
```

![自定义模型](https://raw.githubusercontent.com/lemonade-sdk/assets/refs/heads/main/huggingface_course/custom_model.png)

全部完成！现在让我们来试一试。

### 探索其他模型和加速选项

在上面的例子中，Jan-Nano 模型利用 Vulkan 加速在 AMD GPU 上实现了高效的本地 LLM 推理。您还可以通过访问 http://localhost:8000/#model-management 或查看[模型文档](https://lemonade-server.ai/docs/server/server_models/)来尝试其他模型和加速选项。

对于需要简洁上下文并能从 NPU + iGPU 加速中受益的 Windows 应用，您可以尝试 Lemonade Server 提供的混合模型——该模型针对 AMD Ryzen AI 300 系列 PC 进行了优化。诸如 `Llama-xLAM-2-8b-fc-r-Hybrid` 之类的模型专门为工具调用进行了微调，并能提供快速、响应迅速的性能！
