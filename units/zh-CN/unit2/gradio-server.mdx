# 构建 Gradio MCP 服务器

在本节中，我们将使用 Gradio 创建情感分析 MCP 服务器。该服务器将公开一个情感分析工具，既可以通过 Web 界面供人类用户使用，也可以通过 MCP 协议供 AI 模型使用。

## Gradio MCP 集成简介

Gradio 提供了一种简单直接的方式来创建 MCP 服务器，它会自动将你的 Python 函数转换为 MCP 工具。当你在 `launch()` 中设置 `mcp_server=True` 时，Gradio 会：

1. 自动将你的函数转换为 MCP 工具
2. 将输入组件映射到工具参数模式
3. 从输出组件确定响应格式
4. 设置基于 HTTP+SSE 的 JSON-RPC 进行客户端-服务器通信
5. 同时创建 Web 界面和 MCP 服务器端点

## 设置项目

首先，让我们为项目创建一个新目录并设置所需的依赖项：

```bash
mkdir mcp-sentiment
cd mcp-sentiment
python -m venv venv
source venv/bin/activate  # Windows 系统使用：venv\Scripts\activate
pip install "gradio[mcp]" textblob
```

## 创建服务器

> Hugging Face Spaces 需要一个 app.py 文件来构建空间。因此 Python 文件的名称必须是 app.py

创建一个名为 `app.py` 的新文件，包含以下代码：

```python
import json
import gradio as gr
from textblob import TextBlob

def sentiment_analysis(text: str) -> str:
    """
    分析给定文本的情感。

    Args:
        text (str): 要分析的文本

    Returns:
        str: 包含极性、主观性和评估的 JSON 字符串
    """
    blob = TextBlob(text)
    sentiment = blob.sentiment
    
    result = {
        "polarity": round(sentiment.polarity, 2),  # -1（负面）到 1（正面）
        "subjectivity": round(sentiment.subjectivity, 2),  # 0（客观）到 1（主观）
        "assessment": "positive" if sentiment.polarity > 0 else "negative" if sentiment.polarity < 0 else "neutral"
    }

    return json.dumps(result)

# 创建 Gradio 界面
demo = gr.Interface(
    fn=sentiment_analysis,
    inputs=gr.Textbox(placeholder="输入要分析的文本..."),
    outputs=gr.Textbox(),  # 从 gr.JSON() 改为 gr.Textbox()
    title="文本情感分析",
    description="使用 TextBlob 分析文本的情感"
)

# 启动界面和 MCP 服务器
if __name__ == "__main__":
    demo.launch(mcp_server=True)
```

## 理解代码

让我们分解关键组件：

1. **函数定义**：
   - `sentiment_analysis` 函数接收文本输入并返回 JSON 字典的字符串表示
   - 它使用 TextBlob 来分析情感
   - 文档字符串至关重要，因为它帮助 Gradio 生成 MCP 工具模式
   - 类型提示（`str` 和 `dict`）有助于定义输入/输出模式

2. **Gradio 界面**：
   - `gr.Interface` 同时创建 Web UI 和 MCP 服务器
   - 该函数会自动作为 MCP 工具公开
   - 输入和输出组件定义工具的模式
   - JSON 输出组件确保正确的序列化

3. **MCP 服务器**：
   - 设置 `mcp_server=True` 启用 MCP 服务器
   - 服务器将在 `http://localhost:7860/gradio_api/mcp/sse` 可用
   - 你也可以使用环境变量启用它：
     ```bash
     export GRADIO_MCP_SERVER=True
     ```

## 运行服务器

通过运行以下命令启动服务器：

```bash
python app.py
```

你应该会看到输出，表明 Web 界面和 MCP 服务器都在运行。Web 界面将在 `http://localhost:7860` 可用，MCP 服务器在 `http://localhost:7860/gradio_api/mcp/sse` 可用。

## 测试服务器

你可以通过两种方式测试服务器：

1. **Web 界面**：
   - 在浏览器中打开 `http://localhost:7860`
   - 输入一些文本并点击"提交"
   - 你应该会看到情感分析结果

2. **MCP 模式**：
   - 访问 `http://localhost:7860/gradio_api/mcp/schema`
   - 这显示了客户端将使用的 MCP 工具模式
   - 你也可以在 Gradio 应用页脚的"查看 API"链接中找到它

## 故障排除提示

1. **类型提示和文档字符串**：
   - 始终为函数参数和返回值提供类型提示
   - 为每个参数包含带有"Args:"块的文档字符串
   - 这有助于 Gradio 生成准确的 MCP 工具模式

2. **字符串输入**：
   - 如有疑问，将输入参数接受为 `str`
   - 在函数内部将它们转换为所需类型
   - 这提供了与 MCP 客户端更好的兼容性

3. **SSE 支持**：
   - 某些 MCP 客户端不支持基于 SSE 的 MCP 服务器
   - 在这种情况下，使用 `mcp-remote`：
     ```json
     {
       "mcpServers": {
         "gradio": {
           "command": "npx",
           "args": [
             "mcp-remote",
             "http://localhost:7860/gradio_api/mcp/sse"
           ]
         }
       }
     }
     ```

4. **连接问题**：
   - 如果遇到连接问题，尝试重启客户端和服务器
   - 检查服务器是否正在运行且可访问
   - 验证 MCP 模式在预期的 URL 可用

## 部署到 Hugging Face Spaces

要让你的服务器对其他人可用，你可以将其部署到 Hugging Face Spaces：

1. 在 Hugging Face 上创建一个新的 Space：
   - 访问 [huggingface.co/spaces](https://huggingface.co/spaces)
   - 点击"New Space"
   - 命名你的空间（例如，"mcp-sentiment"）
   - 选择"Gradio"作为 SDK
   - 点击"Create Space"

2. 创建 `requirements.txt` 文件：
```txt
gradio[mcp]
textblob
```

3. 将代码推送到 Space：
```bash
git init
git add app.py requirements.txt
git commit -m "Initial commit"
git remote add origin https://huggingface.co/spaces/YOUR_USERNAME/mcp-sentiment
git push -u origin main
```

你的 MCP 服务器现在将在以下地址可用：
```
https://YOUR_USERNAME-mcp-sentiment.hf.space/gradio_api/mcp/sse
```
