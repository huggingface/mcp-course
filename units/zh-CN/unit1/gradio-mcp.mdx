# Gradio 与 MCP 的集成

我们已经探讨了 MCP 协议的核心概念，以及如何实现 MCP 服务器与客户端。在本节中，我们将学习如何使用 Gradio 轻松地创建 MCP 服务器。

> [!TIP]
> Gradio 是一个流行的 Python 库，可以为机器学习模型快速创建可定制的 Web 界面。

## Gradio 简介

Gradio 让开发者仅需几行 Python 代码，就能为他们的模型创建用户界面。它尤其适用于以下场景：

- 创建演示和原型
- 与非技术背景的用户分享模型
- 测试和调试模型行为

随着 MCP 支持的引入，Gradio 现在提供了一种直接的方式，通过标准化的 MCP 协议来开放 AI 模型的能力。

将 Gradio 与 MCP 相结合，您可以用最少的代码创建出既对人类友好、又可供 AI 访问的工具。更重要的是，Gradio 在 AI 社区中已被广泛使用，因此您可以很方便地与他人分享您的 MCP 服务器。

## 前置条件

要使用支持 MCP 的 Gradio，您需要安装带有 MCP 扩展的 Gradio：

```bash
uv pip install "gradio[mcp]"
```

您还需要一个支持通过 MCP 协议进行工具调用的 LLM 应用，例如 Cursor (我们称之为 “MCP 宿主”)。

## 使用 Gradio 创建 MCP 服务器

让我们通过一个基本示例来演示如何使用 Gradio 创建 MCP 服务器：

```python
import gradio as gr

def letter_counter(word: str, letter: str) -> int:
    """
    统计一个字母在单词或文本中出现的次数。

    Args:
        word (str): 需要检索的输入文本。
        letter (str): 需要计数的字母。

    Returns:
        int: 字母在文本中出现的次数。
    """
    word = word.lower()
    letter = letter.lower()
    count = word.count(letter)
    return count

# 创建一个标准的 Gradio 界面
demo = gr.Interface(
    fn=letter_counter,
    inputs=["textbox", "textbox"],
    outputs="number",
    title="字母计数器",
    description="输入一段文本和一个字母，应用将统计该字母在文本中出现的次数。"
)

# 同时启动 Gradio Web 界面和 MCP 服务器
if __name__ == "__main__":
    demo.launch(mcp_server=True)
```

通过以上设置，您的 `letter_counter` 函数现在可以通过两种方式访问：

1.  传统的 Gradio Web 界面，用于直接的人机交互。
2.  一个可以连接到兼容客户端的 MCP 服务器。

MCP 服务器的访问地址为：
```
http://your-server:port/gradio_api/mcp/sse
```

应用本身依然可以通过 Web 界面访问，它看起来是这样的：

## 幕后原理

当您在 `launch()` 中设置 `mcp_server=True` 时，Gradio 会在后台完成以下工作：

1.  Gradio 函数被自动转换为 MCP 工具。
2.  输入组件被映射到工具的参数模式 (schema)。
3.  输出组件决定了响应的格式。
4.  Gradio 服务器开始监听 MCP 协议消息。
5.  基于 HTTP+SSE 的 JSON-RPC 被设置为客户端与服务器之间的通信方式。

## Gradio 与 MCP 集成的关键特性

1.  **工具转换**：Gradio 应用中的每个 API 端点都会被自动转换为一个 MCP 工具，并带有相应的名称、描述和输入模式。要查看这些工具和模式，请访问 `http://your-server:port/gradio_api/mcp/schema`，或在 Gradio 应用页脚点击 “View API” 链接，然后选择 “MCP”。

2.  **环境变量支持**：您可以通过两种方式启用 MCP 服务器功能：
    -   在 `launch()` 中使用 `mcp_server` 参数：
        ```python
        demo.launch(mcp_server=True)
        ```
    -   使用环境变量：
        ```bash
        export GRADIO_MCP_SERVER=True
        ```

3.  **文件处理**：服务器会自动处理文件数据的转换，包括：
    -   将 base64 编码的字符串转换为文件数据。
    -   处理图像文件并以正确的格式返回。
    -   管理临时文件的存储。

    **强烈建议** 将输入的图像和文件作为完整的 URL (“http://...” 或 “https://...”) 传递，因为 MCP 客户端有时无法正确处理本地文件。

4.  **在 🤗 Spaces 上托管 MCP 服务器**：您可以在 Hugging Face Spaces 上免费发布您的 Gradio 应用，从而拥有一个免费托管的 MCP 服务器。这是一个示例 Space：https://huggingface.co/spaces/abidlabs/mcp-tools

## 故障排查技巧

1.  **类型提示和文档字符串**：请确保为您的函数提供准确的类型提示和格式正确的文档字符串。文档字符串应包含一个 “Args:” 部分，其中参数名称需要缩进。

2.  **字符串输入**：如果不确定，可以将输入参数的类型接受为 `str`，然后在函数内部进行类型转换。

3.  **SSE 支持**：某些 MCP 宿主不支持基于 SSE 的 MCP 服务器。在这种情况下，您可以使用 `mcp-remote` 进行桥接：
    ```json
    {
      "mcpServers": {
        "gradio": {
          "command": "npx",
          "args": [
            "mcp-remote",
            "http://your-server:port/gradio_api/mcp/sse"
          ]
        }
      }
    }
    ```

4.  **重启**：如果遇到连接问题，请尝试重启 MCP 客户端和 MCP 服务器。
