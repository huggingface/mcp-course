## 理解 MCP 客户端

MCP 客户端是充当 AI 应用程序（主机）和 MCP 服务器提供的外部功能之间桥梁的关键组件。可以将主机视为你的主应用程序（如 AI 助手或 IDE），而客户端则是该主机内负责处理 MCP 通信的专用模块。

## 用户界面客户端

让我们首先探索可用于 MCP 的用户界面客户端。

### 聊天界面客户端

- Claude Desktop（Anthropic）

### 交互式开发客户端

- 带有 MCP 的 VS Code 扩展（例如 Continue）
- Cursor IDE（内置 MCP 客户端）
- Zed 编辑器

这些客户端支持连接到多个 MCP 服务器并实时调用工具。

## 快速连接到 Hugging Face MCP 服务器

Hugging Face 提供了一个托管的 MCP 服务器，内置了用于探索模型、数据集、Spaces 和论文的工具。

1. 登录后访问 https://huggingface.co/settings/mcp。
2. 选择你的 MCP 兼容客户端（例如 VS Code、Cursor、Zed、Claude Desktop）。
3. 将生成的配置片段复制到客户端的 MCP 配置中。
4. 重启或重新加载你的客户端。你应该会看到"Hugging Face"已连接。

提示：优先使用生成的片段而不是手写配置；它是针对每个客户端定制的。

## 配置 MCP 客户端

现在我们已经介绍了 MCP 协议的核心，让我们看看如何配置你的 MCP 服务器和客户端。

有效部署 MCP 服务器和客户端需要正确的配置。

> [!TIP]
> MCP 规范仍在不断发展，因此配置方法可能会发生变化。我们将重点关注当前的配置最佳实践。

### MCP 配置文件

MCP 主机使用配置文件来管理服务器连接。这些文件定义了哪些服务器可用以及如何连接到它们。

幸运的是，配置文件非常简单，易于理解，并且在主要的 MCP 主机之间保持一致。

#### `mcp.json` 结构

MCP 的标准配置文件名为 `mcp.json`。以下是基本结构：

这是 `mcp.json` 的基本结构，可以传递给 Claude Desktop、Cursor 或 VS Code 等应用程序。

```json
{
  "servers": [
    {
      "name": "Server Name",
      "transport": {
        "type": "stdio|sse",
        // 传输特定配置
      }
    }
  ]
}
```

在这个示例中，我们有一个带有名称和传输类型的单个服务器。传输类型是 `stdio` 或 `sse`。

#### stdio 传输的配置

对于使用 stdio 传输的本地服务器，配置包括启动服务器进程的命令和参数：

```json
{
  "servers": [
    {
      "name": "File Explorer",
      "transport": {
        "type": "stdio",
        "command": "python",
        "args": ["/path/to/file_explorer_server.py"] // 这是一个示例，我们将在下一单元中使用真实服务器
      }
    }
  ]
}
```

在这里，我们有一个名为"File Explorer"的服务器，它是一个本地脚本。

#### HTTP+SSE 传输的配置

对于使用 HTTP+SSE 传输的远程服务器，配置包括服务器 URL：

```json
{
  "servers": [
    {
      "name": "Remote API Server",
      "transport": {
        "type": "sse",
        "url": "https://example.com/mcp-server"
      }
    }
  ]
}
```

#### 配置中的环境变量

可以使用 `env` 字段将环境变量传递给服务器进程。以下是如何在服务器代码中访问它们：

在 Python 中，我们使用 `os` 模块来访问环境变量：

```python
import os

# 访问环境变量
github_token = os.environ.get("GITHUB_TOKEN")
if not github_token:
    raise ValueError("GITHUB_TOKEN environment variable is required")

# 在服务器代码中使用令牌
def make_github_request():
    headers = {"Authorization": f"Bearer {github_token}"}
    # ... 其余代码
```

## Tiny Agents 客户端

现在，让我们探索如何在代码中使用 MCP 客户端。

你还可以使用 tiny agents 作为 MCP 客户端，直接从代码连接到 MCP 服务器。Tiny agents 提供了一种简单的方法来创建可以使用 MCP 服务器工具的 AI 代理。

Tiny Agent 可以在命令行环境中运行 MCP 服务器。为此，我们需要安装 `npm` 并使用 `npx` 运行服务器。**Python 和 JavaScript 都需要这些。**

让我们使用 `npm` 安装 `npx`。如果你没有安装 `npm`，请查看 [npm 文档](https://docs.npmjs.com/downloading-and-installing-node-js-and-npm)。

### 设置

首先，如果你没有安装 `npx`，我们需要安装它。你可以使用以下命令：

```bash
# 安装 npx
npm install -g npx
```

然后，我们需要安装带有 MCP 支持的 huggingface_hub 包。这将允许我们运行 MCP 服务器和客户端。

```bash
pip install "huggingface_hub[mcp]>=0.32.0"
```

然后，我们需要登录到 Hugging Face Hub 以访问 MCP 服务器。你可以使用 `huggingface-cli` 命令行工具来完成此操作。你需要一个[登录令牌](https://huggingface.co/docs/huggingface_hub/v0.32.3/en/quick-start#authentication)来执行此操作。

```bash
huggingface-cli login
```

### 配置访问令牌权限

创建 Hugging Face 访问令牌并登录后，你需要确保令牌具有与推理提供商一起工作的适当权限。

> [!WARNING]
> **重要：** 如果跳过此步骤，在使用托管模型运行 tiny agents 时可能会遇到身份验证错误。

1. 访问你的 [Hugging Face 访问令牌页面](https://huggingface.co/settings/tokens)
2. 找到你的 MCP 令牌并点击旁边的三个点（⋮）
3. 选择**"编辑权限"**
4. 在**推理**部分下，勾选以下选项：
   - **"调用推理提供商"**
5. 保存更改

此权限是必需的，因为 tiny agents 需要通过 Nebius 等提供商对托管模型（如 `Qwen/Qwen2.5-72B-Instruct`）进行 API 调用。

### 连接到 MCP 服务器

现在，让我们创建一个代理配置文件 `agent.json`。

```json
{
    "name": "playwright-agent",
    "description": "Agent with Playwright MCP server",
    "model": "Qwen/Qwen2.5-72B-Instruct",
    "provider": "nebius",
    "servers": [
        {
            "type": "stdio",
            "command": "npx",
            "args": ["@playwright/mcp@latest"]
        }
    ]
}
```

在这个配置中，我们使用 `@playwright/mcp` MCP 服务器。这是一个可以使用 Playwright 控制浏览器的 MCP 服务器。

现在你可以运行代理：

```bash
tiny-agents run agent.json
```
