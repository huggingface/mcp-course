## 理解 MCP 客户端

MCP 客户端是连接 AI 应用 (宿主) 与 MCP 服务器所提供的外部功能的关键组件。您可以将宿主视为您的主应用程序 (例如 AI 助手或 IDE)，而客户端则是该宿主内部负责处理 MCP 通信的专用模块。

## 用户界面客户端

让我们首先来探索一些可用的图形化 MCP 客户端。

### 聊天界面客户端

-   Claude Desktop (Anthropic)

### 交互式开发客户端

-   支持 MCP 的 VS Code 扩展 (例如 Continue)
-   Cursor IDE (内置 MCP 客户端)
-   Zed 编辑器

这些客户端都支持同时连接到多个 MCP 服务器，并实时调用它们提供的工具。

## 快速连接到 Hugging Face MCP 服务器

Hugging Face 提供了一个托管的 MCP 服务器，其中内置了用于探索模型、数据集、Space 和论文的工具。

1.  登录后访问 https://huggingface.co/settings/mcp。
2.  选择您正在使用的、兼容 MCP 的客户端 (例如 VS Code、Cursor、Zed、Claude Desktop)。
3.  将生成的配置代码片段复制到客户端的 MCP 配置文件中。
4.  重启或重新加载您的客户端。您应该能看到 “Hugging Face” 出现在已连接的服务器列表中。

提示：请优先使用自动生成的代码片段，而不是手动编写配置，因为它已针对每个客户端进行了适配。

## 配置 MCP 客户端

现在我们已经了解了 MCP 协议的核心，让我们看看如何配置您的 MCP 服务器和客户端。

要有效地部署 MCP 服务器和客户端，正确的配置至关重要。

> [!TIP]
> MCP 规范仍在不断发展，因此配置方法将来可能会发生变化。我们将重点关注当前推荐的最佳实践。

### MCP 配置文件

MCP 宿主使用配置文件来管理服务器连接。这些文件定义了哪些服务器可用，以及如何连接到它们。

幸运的是，这些配置文件非常简单直观，并且在主流的 MCP 宿主之间保持着良好的一致性。

#### `mcp.json` 结构

MCP 的标准配置文件名为 `mcp.json`。以下是其基本结构，可用于 Claude Desktop、Cursor 或 VS Code 等应用。

```json
{
  "servers": [
    {
      "name": "Server Name",
      "transport": {
        "type": "stdio|sse",
        // 针对不同传输方式的特定配置
      }
    }
  ]
}
```

在这个示例中，我们定义了一个服务器，包含其名称和传输类型 (`stdio` 或 `sse`)。

#### stdio 传输配置

对于使用 stdio 传输的本地服务器，配置中需要包含启动服务器进程的命令和参数：

```json
{
  "servers": [
    {
      "name": "File Explorer",
      "transport": {
        "type": "stdio",
        "command": "python",
        "args": ["/path/to/file_explorer_server.py"] // 这是一个示例，我们将在下一单元中使用真实的服务器
      }
    }
  ]
}
```

在这里，我们定义了一个名为 “File Explorer” 的服务器，它是一个本地的 Python 脚本。

#### HTTP+SSE 传输配置

对于使用 HTTP+SSE 传输的远程服务器，配置中需要包含服务器的 URL：

```json
{
  "servers": [
    {
      "name": "Remote API Server",
      "transport": {
        "type": "sse",
        "url": "https://example.com/mcp-server"
      }
    }
  ]
}
```

#### 在配置中设置环境变量

您可以使用 `env` 字段将环境变量传递给服务器进程。以下是在服务器代码中访问它们的方法：

在 Python 中，我们使用 `os` 模块来访问环境变量：

```python
import os

# 访问环境变量
github_token = os.environ.get("GITHUB_TOKEN")
if not github_token:
    raise ValueError("需要设置 GITHUB_TOKEN 环境变量")

# 在服务器代码中使用 token
def make_github_request():
    headers = {"Authorization": f"Bearer {github_token}"}
    # ... 其余代码
```

## Tiny Agents 客户端

现在，让我们来探索如何在代码中使用 MCP 客户端。

您还可以使用 Tiny Agents 作为 MCP 客户端，直接从代码中连接到 MCP 服务器。Tiny Agents 提供了一种简单的方法来创建可以使用 MCP 服务器工具的 AI 代理。

Tiny Agents 可以在命令行环境中运行 MCP 服务器。为此，我们需要安装 `npm` 并使用 `npx` 来运行服务器。**无论您使用 Python 还是 JavaScript，都需要这两个工具。**

让我们使用 `npm` 来安装 `npx`。如果您尚未安装 `npm`，请查阅 [npm 文档](https://docs.npmjs.com/downloading-and-installing-node-js-and-npm)。

### 环境设置

首先，如果您没有安装 `npx`，请使用以下命令进行安装：

```bash
# 安装 npx
npm install -g npx
```

然后，我们需要安装带有 MCP 支持的 `huggingface_hub` 包。这将允许我们运行 MCP 服务器和客户端。

```bash
pip install "huggingface_hub[mcp]>=0.32.0"
```

接下来，我们需要登录到 Hugging Face Hub 以访问 MCP 服务器。您可以使用 `huggingface-cli` 命令行工具来完成此操作。您需要一个[登录令牌](https://huggingface.co/docs/huggingface_hub/v0.32.3/en/quick-start#authentication)来执行此操作。

```bash
huggingface-cli login
```

### 配置访问令牌权限

创建 Hugging Face 访问令牌并登录后，您需要确保该令牌具有与推理服务提供商交互的适当权限。

> [!WARNING]
> **重要提示：** 如果跳过此步骤，在使用托管模型运行 `tiny-agents` 时可能会遇到身份验证错误。

1.  访问您的 [Hugging Face 访问令牌页面](https://huggingface.co/settings/tokens)。
2.  找到您的 MCP 令牌，然后点击旁边的三个点 (⋮)。
3.  选择 **“编辑权限”**。
4.  在 **推理** 部分下，勾选以下选项：
    -   **“调用推理提供商”**
5.  保存更改。

此权限是必需的，因为 Tiny Agents 需要通过 Nebius 等提供商对托管模型 (例如 `Qwen/Qwen2.5-72B-Instruct`) 进行 API 调用。

### 连接到 MCP 服务器

现在，让我们创建一个名为 `agent.json` 的代理配置文件。

```json
{
    "name": "playwright-agent",
    "description": "一个集成了 Playwright MCP 服务器的代理",
    "model": "Qwen/Qwen2.5-72B-Instruct",
    "provider": "nebius",
    "servers": [
        {
            "type": "stdio",
            "command": "npx",
            "args": ["@playwright/mcp@latest"]
        }
    ]
}
```

在此配置中，我们使用了 `@playwright/mcp` 这个 MCP 服务器，它可以使用 Playwright 来控制浏览器。

现在，您可以运行该代理了：

```bash
tiny-agents run agent.json
```
